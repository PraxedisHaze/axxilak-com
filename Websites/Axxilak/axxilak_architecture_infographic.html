Why This Architecture?

Traditional site builders trap users in subscriptions. The Axxilak model prioritizes ownership and speed. By eliminating server-side dependencies for the editor, we reduce friction and cost.

  • Zero Friction: No login required to start editing.
  • Persistence: `localStorage` saves progress automatically.
  • Portability: The output is a simple HTML file.

Technical Constraints

To maintain speed and portability, we operate under strict limits. The entire standalone editor bundle must remain lightweight.

No Server-Side Code

Logic must rely on `FileReader` API and `Blob` generation. No databases.

Security Sanitization

Must use `DOMPurify` to clean user input before injecting into the DOM to prevent XSS.

Image Optimization

Images > 500KB trigger a warning to prevent bloated Base64 strings.
